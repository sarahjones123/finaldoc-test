<h1>Design Studio</h1>
<p>Design Studio is where you set your knowledge base's visual identity — brand colors, typography, and theme. Changes are previewed in real time and apply to your entire public knowledge base when saved.</p>

<h2>Opening Design Studio</h2>
<ol>
<li>Go to <strong>Settings</strong> in the left sidebar</li>
<li>Click <strong>Appearance</strong> → <strong>Design Studio</strong></li>
</ol>

<h2>Brand Colors</h2>
<p>FinalDoc uses six CSS color variables that control the entire KB appearance. Set each one by clicking the color swatch and using the color picker (or entering a hex code):</p>

<table>
<thead><tr><th>Variable</th><th>What It Controls</th><th>Example Value</th></tr></thead>
<tbody>
<tr><td><strong>Primary</strong></td><td>Links, buttons, active items, headings highlights, search bar</td><td>#3b82f6 (blue)</td></tr>
<tr><td><strong>Secondary</strong></td><td>Hover states, secondary accents, badges</td><td>#6366f1 (indigo)</td></tr>
<tr><td><strong>Accent</strong></td><td>Special highlights, featured items, AI chatbot bubble</td><td>#8b5cf6 (purple)</td></tr>
<tr><td><strong>Background</strong></td><td>Main page background, sidebar background</td><td>#ffffff (white)</td></tr>
<tr><td><strong>Text</strong></td><td>Main body text color</td><td>#111827 (dark grey)</td></tr>
<tr><td><strong>Border</strong></td><td>Card borders, dividers, input field borders</td><td>#e5e7eb (light grey)</td></tr>
</tbody>
</table>

<h2>Typography</h2>
<ul>
<li><strong>Heading font</strong> — Choose from Google Fonts for article and page headings</li>
<li><strong>Body font</strong> — Choose from Google Fonts for paragraph text, labels, and UI elements</li>
<li><strong>Code font</strong> — The monospace font used in code blocks (default: JetBrains Mono)</li>
<li><strong>Font size</strong> — Base body font size (default: 16px)</li>
</ul>

<h2>Theme Presets</h2>
<p>Start with a preset theme and customize from there:</p>
<ul>
<li><strong>Default</strong> — Clean blue and white, professional look</li>
<li><strong>Dark</strong> — Dark grey background with light text</li>
<li><strong>Minimal</strong> — Almost no color, maximum white space</li>
<li><strong>Warm</strong> — Warm amber and cream tones</li>
<li><strong>Tech</strong> — Deep dark background with neon accent colors</li>
</ul>
<p>Selecting a preset fills in all six color variables at once. You can then modify individual colors to match your brand.</p>

<h2>Dark Mode Variants</h2>
<p>You can configure separate color sets for light mode and dark mode. Toggle the <strong>Dark Mode Preview</strong> switch to see and edit the dark mode appearance. The KB automatically shows the right variant based on the reader's OS preference.</p>

<h2>Logo and Favicon</h2>
<ul>
<li><strong>Logo</strong> — Appears in the KB header. Upload PNG (recommended), SVG, or JPG. For best results, use a horizontal logo on a transparent background</li>
<li><strong>Dark mode logo</strong> — A logo variant optimized for dark backgrounds (optional)</li>
<li><strong>Favicon</strong> — The tiny icon in browser tabs, 32×32 pixels or 64×64 pixels PNG or ICO format</li>
</ul>

<h2>Live Preview</h2>
<p>As you make changes, the right side of the screen shows a live preview of your KB homepage, an article page, and the search bar. Switch between views using the tabs at the top of the preview panel. Click <strong>Open Full Preview</strong> to see a full-screen preview in a new tab before saving.</p>

<h2>Saving</h2>
<p>Click <strong>Save Changes</strong> to apply your design to the live KB. Changes are instant — no build step or cache clearing needed.</p>
