<h1>Site Visibility</h1>
<p>Site Visibility controls who can access your public knowledge base. FinalDoc offers three visibility modes so you can match your audience and security requirements.</p>

<h2>Opening Site Visibility</h2>
<ol>
<li>Go to <strong>Settings → Security &amp; Access → Site Visibility</strong></li>
</ol>

<h2>Visibility Modes</h2>

<h3>Public</h3>
<p>Anyone can visit your knowledge base and read all articles without logging in. This is the default and is appropriate for most public product documentation, help centers, and developer documentation.</p>

<h3>Private</h3>
<p>All content requires reader login. Visitors who are not logged in see only the login/registration page. Use Private for internal documentation (employee handbook, internal SOPs) or customer-only resources.</p>

<h3>Mixed</h3>
<p>Some content is public (no login required) and some content is private (requires login). Visitors can browse public articles freely. When they try to access a private article or category, they are redirected to the login page.</p>
<p>After selecting Mixed, configure which categories or articles are private in <strong>Settings → Security &amp; Access → Content Access &amp; Mixed Visibility</strong>.</p>

<h2>Changing Visibility</h2>
<ol>
<li>Select the mode you want</li>
<li>Click <strong>Save</strong></li>
<li>The change takes effect immediately for all visitors</li>
</ol>

<h2>Impact on Reader Login</h2>
<ul>
<li><strong>Public</strong> — Reader login button can be hidden (no reason to log in if all content is public)</li>
<li><strong>Private</strong> — All visitors must log in. If self-registration is disabled, readers need an invite from you</li>
<li><strong>Mixed</strong> — Login button is always visible so readers can log in to access gated content</li>
</ul>

<h2>Impact on Search Engine Indexing</h2>
<ul>
<li><strong>Public</strong> — All articles can be indexed by search engines (Google, Bing)</li>
<li><strong>Private</strong> — The robots.txt disallows all crawling. Private content stays off search engines</li>
<li><strong>Mixed</strong> — Public articles are indexed, private articles are not</li>
</ul>
