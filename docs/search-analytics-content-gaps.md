<h1>Search Analytics and Content Gap Analysis</h1>
<p>Search analytics shows you what your readers are looking for — and what they cannot find. Every search query entered in your knowledge base is logged. Use this data to identify content gaps, improve existing articles, and create new ones that address real reader needs.</p>

<h2>Opening Search Analytics</h2>
<ol>
<li>Click <strong>Analytics</strong> in the left sidebar</li>
<li>Click <strong>Search Analytics &amp; Content Gaps</strong></li>
</ol>

<h2>Understanding the Search Analytics Table</h2>
<p>The table shows each unique search query with:</p>
<ul>
<li><strong>Query</strong> — The exact search term the reader typed</li>
<li><strong>Count</strong> — How many times this query was searched</li>
<li><strong>Results found</strong> — How many articles were returned for this query</li>
<li><strong>Zero results flag</strong> — Highlighted in red if the query returned no articles</li>
<li><strong>Last searched</strong> — When this query was most recently searched</li>
</ul>

<h2>Content Gaps (Zero-Result Searches)</h2>
<p>The most valuable part of Search Analytics is the zero-result queries — these are searches where your readers found nothing. Each zero-result query represents a content gap: a topic your readers need help with that is not covered (or not well-covered) in your knowledge base.</p>
<p>To see only content gaps, filter by <strong>Zero Results Only</strong>.</p>

<h2>Fixing Content Gaps</h2>
<p>For each zero-result query, you have two options:</p>

<h3>Option 1: Auto-Generate a Draft Article</h3>
<ol>
<li>Click the <strong>Generate Draft</strong> button next to any content gap query</li>
<li>FinalDoc's AI writes a draft article based on the query topic</li>
<li>The draft article is created in your Documentation workspace</li>
<li>Open it, review and edit the AI-generated content, and publish when ready</li>
</ol>

<h3>Option 2: Create Manually or Update an Existing Article</h3>
<ul>
<li>If you already have an article that should be answering this query but is not showing up in search, open that article and add the search term's keywords to the title, description, or content</li>
<li>Or create a new article from scratch in Documentation and target the query's topic</li>
</ul>

<h2>High-Volume Searches</h2>
<p>Pay attention to queries that appear many times — even if they returned results, if the same thing is being searched repeatedly, it might mean:</p>
<ul>
<li>The existing article is hard to find (navigation or title issue)</li>
<li>The existing article does not fully answer the question (content quality issue)</li>
<li>The topic deserves a more prominent place in your KB (featured or pinned article)</li>
</ul>

<h2>Search Trends Over Time</h2>
<p>Use the date range filter to compare search patterns over different time periods. A query that suddenly spikes in frequency often correlates with:</p>
<ul>
<li>A new product release that introduced something confusing</li>
<li>A support ticket trend that readers are now self-serving</li>
<li>A marketing campaign that brought new visitors who have different questions</li>
</ul>
