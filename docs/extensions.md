<h1>Extensions</h1>
<p>FinalDoc's Extensions let you connect your knowledge base with the other tools your team already uses — help desks, team chat apps, code repositories, and translation platforms. Extensions are organized into three dedicated tabs in the Settings sidebar.</p>

<h2>Where to Find Extensions</h2>
<p>Extensions are split across three sidebar tabs under <strong>Settings → Integrations</strong>:</p>
<table>
<thead><tr><th>Tab</th><th>What's Inside</th></tr></thead>
<tbody>
<tr><td><strong>Chat &amp; Support</strong></td><td>AI Agent (OpenClaw), help desks (Freshdesk, Zendesk, Intercom), team collaboration (Slack, Teams, Drift)</td></tr>
<tr><td><strong>Code Repositories</strong></td><td>GitHub, GitLab, Bitbucket — two-way docs-as-code sync</td></tr>
<tr><td><strong>Translation</strong></td><td>Crowdin, Phrase, Chrome browser extension</td></tr>
</tbody>
</table>

<h2>Help Desks</h2>
<p>Connect FinalDoc with customer support platforms so your support agents can search and share KB articles directly from their helpdesk tickets:</p>
<ul>
<li><strong>Freshdesk</strong> — Search and send articles in customer conversations, create new articles from support requests</li>
<li><strong>Freshservice</strong> — Enhance ticket conversations with contextual articles and internal knowledge</li>
<li><strong>Zendesk</strong> — Search, share, and create knowledge base articles from Zendesk tickets</li>
<li><strong>Intercom</strong> — Suggest and share articles directly from Intercom conversations</li>
<li><strong>Salesforce</strong> — Create, search, and send articles to customers within Salesforce (enterprise plan)</li>
</ul>

<h2>Team Collaboration</h2>
<p>Connect FinalDoc with your team's communication platforms:</p>
<ul>
<li><strong>Slack</strong> — Use <code>/finaldoc</code> commands in Slack to search your KB, get article summaries, and receive notifications when articles are published. See the Slack integration article for setup details</li>
<li><strong>Microsoft Teams</strong> — Ask questions and get AI-powered answers from your KB directly in Teams channels using <code>@FinalDoc</code> mentions. See the <a href="/article/microsoft-teams-integration">Microsoft Teams integration</a> article for setup details</li>
<li><strong>Drift</strong> — Automatically suggest docs to customers in Drift conversations</li>
</ul>

<h2>Code Repositories</h2>
<div style="background:#fef3c7;border-left:4px solid #f59e0b;padding:12px 16px;border-radius:4px;margin:16px 0">
  <strong>Code Repositories vs Doc Autopilot:</strong> Code Repositories syncs existing <code>.md</code> files between your repo and FinalDoc (file transfer, no AI). If you want AI to <em>analyze your source code</em> and generate new documentation, use <strong>Doc Autopilot</strong> (sidebar → AI Studio → Doc Autopilot) instead.
</div>

<p>Sync documentation between FinalDoc and your code repository:</p>
<ul>
<li><strong>GitHub</strong> — Two-way sync between FinalDoc articles and a GitHub repo. Import docs from GitHub or export articles back to your repo</li>
<li><strong>GitLab</strong> — Same sync capabilities with GitLab, including self-hosted instances</li>
<li><strong>Bitbucket</strong> — Sync with Bitbucket Cloud repositories</li>
</ul>

<h2>Translation &amp; Browser</h2>
<ul>
<li><strong>Chrome</strong> — Access articles anytime with the Chrome browser extension</li>
<li><strong>Crowdin</strong> — Automate translations with Crowdin and keep content in sync</li>
<li><strong>Phrase</strong> — Translate and manage multilingual documentation in Phrase</li>
</ul>

<h2>Setting Up an Extension</h2>
<ol>
<li>Navigate to the appropriate tab (<strong>Chat &amp; Support</strong>, <strong>Code Repositories</strong>, or <strong>Translation</strong>)</li>
<li>Find the extension you want and click <strong>Setup</strong></li>
<li>Enter the required credentials (API key, webhook URL, access token, etc.)</li>
<li>Click <strong>Save</strong></li>
</ol>
<p>Connected extensions show a green "Connected" badge. You can disconnect at any time by clicking the extension and selecting <strong>Disconnect</strong>.</p>
