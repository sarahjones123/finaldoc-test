<p><strong>FinalDoc</strong> is a modern knowledge base platform designed to help teams create, manage, and publish high-quality documentation from a single, unified workspace. It supports a wide range of documentation needs, including product documentation, API references, help centres, internal knowledge bases, and customer support portals.</p>
<p>With its intuitive interface and powerful feature set, FinalDoc enables teams to streamline content creation, maintain consistency, and deliver a seamless reading experience to end users. It combines structured content management with AI-powered capabilities to improve both authoring efficiency and content discoverability.</p>
<h2>What you can do with FinalDoc</h2>
<p>FinalDoc provides a comprehensive set of features that cover the entire documentation lifecycle, from creation to publishing and analysis.</p>
<h3>Write and edit</h3>
<p>Create and maintain content using a powerful rich-text editor.</p>
<ul>
<li>Supports formatting options such as headings, lists, tables, and images.  </li>
<li>Includes code blocks for technical documentation.  </li>
<li>Provides AI-assisted writing to improve content quality and speed.  </li>
<li>Enables easy editing and version updates.</li>
</ul>
<h3>Organise</h3>
<p>Structure your documentation in a logical and scalable way.</p>
<ul>
<li>Create categories and subcategories to group related content.  </li>
<li>Use drag-and-drop functionality to reorder content.  </li>
<li>Apply folder colours and pin important articles for quick access.  </li>
<li>Maintain a clean and navigable content hierarchy.</li>
</ul>
<h3>Publish</h3>
<p>Control how and where your content is visible.</p>
<ul>
<li>Publish articles publicly for external users or privately for internal teams.  </li>
<li>Configure visibility at the article level.  </li>
<li>Ensure controlled access to sensitive information.</li>
</ul>
<h3>Search</h3>
<p>Enable users to quickly find relevant information.</p>
<ul>
<li>Full-text search across all content.  </li>
<li>AI-powered semantic search for better context-based results.  </li>
<li>Fuzzy search to handle typos and partial matches.  </li>
<li>Improves content discoverability and user satisfaction.</li>
</ul>
<h3>Translate</h3>
<p>Reach a global audience with multilingual support.</p>
<ul>
<li>One-click AI translation into 30+ languages.  </li>
<li>Maintain consistency across translated content.  </li>
<li>Reduce manual effort in localisation.</li>
</ul>
<h3>Analyse</h3>
<p>Gain insights into how your content is used.</p>
<ul>
<li>Track reader behaviour and engagement.  </li>
<li>View search queries to understand user intent.  </li>
<li>Monitor content performance using health scores.  </li>
<li>Collect feedback to improve documentation quality.</li>
</ul>
<h3>Collaborate</h3>
<p>Work efficiently with your team.</p>
<ul>
<li>Add team members with defined roles and permissions.  </li>
<li>Use inline comments and discussions for content review.  </li>
<li>Implement approval workflows for publishing control.  </li>
<li>Ensure accountability and content governance.</li>
</ul>
<h3>Brand</h3>
<p>Customise the look and feel of your knowledge base.</p>
<ul>
<li>Configure custom domains.  </li>
<li>Apply brand logos, colours, and fonts.  </li>
<li>Adjust layout and design settings.  </li>
<li>Deliver a consistent brand experience.</li>
</ul>
<h3>Widget</h3>
<p>Integrate help directly into your product or website.</p>
<ul>
<li>Embed a help widget for instant access to documentation.  </li>
<li>Includes AI chat and voice assistant capabilities.  </li>
<li>Improves self-service support for users.</li>
</ul>
<h3>AI agent</h3>
<p>Extend your knowledge base to external platforms.</p>
<ul>
<li>Connect with platforms such as WhatsApp, Telegram, Slack, Discord, and Teams.  </li>
<li>Enable automated responses using your documentation content.  </li>
<li>Provide support across multiple communication channels.</li>
</ul>
<h2>Key concepts</h2>
<p>Understanding the core components of FinalDoc helps in effectively managing your knowledge base.</p>
<ul>
<li><strong>Project</strong> — Represents your entire knowledge base. Each project has its own URL, branding, and configuration settings.  </li>
<li><strong>Workspace</strong> — A logical section within a project used to organize content. A project can have one or multiple workspaces.  </li>
<li><strong>Category</strong> — A folder structure used to group related articles. Categories can be nested up to three levels.  </li>
<li><strong>Article</strong> — An individual documentation page that contains content. Each article belongs to a category.  </li>
<li><strong>Reader</strong> — An end user who accesses and consumes published documentation.  </li>
<li><strong>Team member</strong> — A user responsible for creating and managing content, assigned with roles such as Owner, Admin, Editor, Draft writer, or Viewer.</li>
</ul>
