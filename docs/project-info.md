<p>The Project Info settings page is where you manage the basic details of your FinalDoc project — its name and the URL slug that identifies it. You'll also find the option to delete the project here (Owners only).</p>
<h2>Opening Project Info</h2>
<ol>
<li>Go to <strong>Settings</strong> in the left sidebar</li>
<li>Click <strong>General → Project Info</strong></li>
</ol>
<h2>Project Name</h2>
<p>The project name appears:</p>
<ul>
<li>In the left sidebar switcher at the top (when you have multiple projects)</li>
<li>As the default KB name on your public knowledge base (unless you've set a custom KB title in Portal Builder)</li>
<li>In browser tabs when readers visit your KB</li>
<li>In email notifications and system messages</li>
</ul>
<p>To change: Edit the <strong>Project Name</strong> field and click <strong>Save</strong>.</p>
<h2>Project URL (Slug)</h2>
<p>The project slug is the URL-friendly identifier used in:</p>
<ul>
<li>Your FinalDoc subdomain URL: <code>https://{slug}.finaldoc.io</code></li>
<li>Internal API references and project identification</li>
</ul>
<p>The slug is set when the project is created and is <strong>not editable after creation</strong> — changing it would break existing links and bookmarks. Contact support if you need a slug change.</p>
<h2>Knowledge Base Language</h2>
<p>Language settings for your knowledge base are managed per workspace, not at the project level. To configure the language for your knowledge base:</p>
<ol>
<li>Go to <strong>Settings → Knowledge Bases</strong></li>
<li>Select your workspace</li>
<li>Update the <strong>Language</strong> setting</li>
</ol>
<p>The language setting affects spell-check behavior in the editor and the HTML <code>lang</code> attribute on your public KB pages (important for accessibility and search engines).</p>
<h2>Deleting the Project</h2>
<p>Permanently deletes the entire project — all articles, categories, settings, and reader data. This cannot be undone.</p>
<ol>
<li>Scroll to the bottom of the Project Info page</li>
<li>Click <strong>Delete Project</strong></li>
<li>Type the project name to confirm</li>
<li>Click <strong>Delete</strong></li>
</ol>
<p><strong>Only OWNER role users can delete a project.</strong> ADMINs cannot see or use this option.</p>
<p>Before deleting: Export your content in <strong>Toolbox → Import / Export</strong> if you want to keep a copy.</p>
