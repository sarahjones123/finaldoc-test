<h1>Folder Colors &amp; Visual Organization</h1>
<p>FinalDoc lets you assign colors and icons to categories (folders) in your Documentation sidebar. This makes it much easier to navigate a large knowledge base at a glance — you can spot "API Reference" (blue) vs. "Getting Started" (green) vs. "Troubleshooting" (red) instantly, without reading every folder name.</p>

<h2>Assigning a Color to a Category</h2>
<ol>
<li>In the <strong>Documentation</strong> view, right-click a category in the sidebar (or click the three-dot menu next to it)</li>
<li>Select <strong>Edit Category</strong></li>
<li>In the category edit dialog, find the <strong>Color</strong> picker</li>
<li>Click a color swatch to select it (10+ preset colors available: red, orange, yellow, green, teal, blue, purple, pink, gray, and more)</li>
<li>Click <strong>Save</strong></li>
</ol>
<p>The category folder in the sidebar immediately changes to the selected color.</p>

<h2>Assigning an Icon to a Category</h2>
<ol>
<li>Open the category edit dialog (right-click → Edit Category)</li>
<li>Find the <strong>Icon</strong> selector</li>
<li>Browse the icon library — dozens of icons are available: book, code, settings gear, lightning bolt, rocket, shield, users, and more</li>
<li>Click an icon to select it</li>
<li>Click <strong>Save</strong></li>
</ol>
<p>The icon replaces the default folder icon in the sidebar.</p>

<h2>Nesting and Visual Hierarchy</h2>
<p>Colors and icons apply to top-level categories and sub-categories. Nested sub-categories inherit the visual style of their parent unless you set a different color/icon on them explicitly. For deeply nested structures, using parent colors only (and leaving sub-categories at default) creates a clean visual hierarchy without overwhelming the sidebar.</p>

<h2>Tips for Visual Organization</h2>
<ul>
<li><strong>Color by audience</strong> — Green = Developers, Blue = End users, Red = Admin/IT</li>
<li><strong>Color by content type</strong> — Blue = Tutorials, Orange = Reference, Red = Troubleshooting</li>
<li><strong>Color by product area</strong> — Each product module gets its own color family</li>
<li><strong>Use icons for quick scanning</strong> — A code icon on API docs, a wrench on troubleshooting, a star on getting started guides</li>
</ul>

<h2>Are Colors Visible to Readers?</h2>
<p>Yes — category colors and icons appear in the sidebar of your public knowledge base as well as in the admin Documentation view. Readers benefit from the visual organization just as your team does.</p>
