<h1>Images, Videos, and Media</h1>
<p>Visuals make documentation easier to understand. FinalDoc makes it easy to add images, videos, and other media directly into your articles — without leaving the editor.</p>

<h2>Adding Images</h2>
<h3>Method 1: Upload from Computer</h3>
<ol>
<li>Click the <strong>Image</strong> button in the editor toolbar (the picture icon)</li>
<li>Click <strong>Upload Image</strong></li>
<li>Select an image file from your computer (JPG, PNG, GIF, WebP, SVG supported)</li>
<li>The image uploads to your Drive and inserts at the cursor position</li>
</ol>

<h3>Method 2: Paste from Clipboard</h3>
<p>If you took a screenshot or copied an image:</p>
<ul>
<li>Click in the editor where you want the image</li>
<li>Press <strong>Ctrl+V / Cmd+V</strong></li>
<li>FinalDoc detects the image on your clipboard and inserts it automatically</li>
</ul>

<h3>Method 3: Drag and Drop</h3>
<ul>
<li>Open your file manager or desktop</li>
<li>Drag an image file directly into the editor area</li>
<li>Drop it at the position you want — it uploads and inserts immediately</li>
</ul>

<h3>Method 4: From Drive</h3>
<ol>
<li>Click the Image button in the toolbar</li>
<li>Switch to the <strong>Drive</strong> tab in the dialog</li>
<li>Browse your uploaded files and click any image to insert it</li>
</ol>

<h2>Image Options</h2>
<p>After inserting an image, click on it to access options:</p>
<ul>
<li><strong>Caption</strong> — Add a caption below the image (shown to readers, helps with accessibility)</li>
<li><strong>Alt text</strong> — A description for screen readers and SEO. Always fill this in for important images</li>
<li><strong>Size</strong> — Drag the corner handles to resize, or choose a preset size (Small / Medium / Large / Full Width)</li>
<li><strong>Alignment</strong> — Align left, center, or right relative to the text</li>
<li><strong>Link</strong> — Make the image clickable by linking it to a URL</li>
</ul>

<h2>Adding Videos</h2>
<h3>YouTube / Vimeo Embed</h3>
<ol>
<li>Copy the video URL from YouTube or Vimeo</li>
<li>In the editor, click the <strong>Embed</strong> button in the toolbar</li>
<li>Paste the video URL</li>
<li>Click <strong>Insert</strong> — the video embeds as a playable player directly in the article</li>
</ol>
<p>Alternatively, paste the YouTube URL directly on an empty line — FinalDoc may auto-detect it and offer to convert it to an embed.</p>

<h3>Video Files from Drive</h3>
<p>If you have a video file uploaded to Drive:</p>
<ol>
<li>Go to <strong>Drive</strong> and find your video file</li>
<li>Make the file public by clicking <strong>Share → Make Public</strong></li>
<li>Copy the public URL</li>
<li>Back in the article editor, use the Embed button and paste the URL</li>
</ol>

<h2>Generated Documentation Videos</h2>
<p>FinalDoc includes a <strong>Video Studio (AI Studio sidebar)</strong> feature in the Toolbox that automatically generates professional documentation videos from your articles. The AI creates a narration script, generates voiceover audio, captures app screenshots, and assembles a full video with transitions and subtitles. See the <strong>Video Generation</strong> articles for a complete guide.</p>

<h2>Uploading Files (Attachments)</h2>
<p>You can attach downloadable files to articles — PDFs, spreadsheets, ZIP files, etc.:</p>
<ol>
<li>Click the <strong>File / Attachment</strong> button in the editor toolbar</li>
<li>Upload the file or select it from Drive</li>
<li>A download link is inserted at the cursor position</li>
<li>Readers see a download button they can click to save the file</li>
</ol>

<h2>Image Storage and Management</h2>
<p>All uploaded images and media are stored in your <strong>Drive</strong> (DigitalOcean Spaces object storage). You can manage them there:</p>
<ul>
<li>See all uploaded files and their storage usage</li>
<li>Rename, move, or delete files</li>
<li>Make files public or private</li>
<li>Copy public URLs for use in other places</li>
</ul>
<p>Images inserted into articles reference their Drive URL. If you delete an image from Drive, it will break in any articles that use it — so be careful before deleting media files.</p>

<h2>Best Practices</h2>
<ul>
<li>Use screenshots to show step-by-step instructions — a picture is worth a thousand words</li>
<li>Compress large images before uploading (PNG files can be large; JPG or WebP are often smaller for photos)</li>
<li>Always add alt text to images so screen readers can describe them to visually impaired readers</li>
<li>Use captions to explain what the image shows, especially for complex diagrams</li>
<li>Avoid images wider than your content area — use the "Full Width" option only when the image needs it</li>
</ul>
