<h1>Tags and SEO Metadata</h1>
<p>The Tags and SEO Metadata tool in the Toolbox lets you view and edit tags and SEO descriptions for all your articles in one place — without opening each article individually. It is much faster for bulk SEO audits and tag management.</p>

<h2>Opening Tags and SEO Metadata</h2>
<ol>
<li>Go to <strong>Toolbox</strong> in the left sidebar</li>
<li>Under <strong>Content</strong>, click <strong>SEO Metadata</strong></li>
</ol>

<h2>The SEO Metadata View</h2>
<p>You see a table with one row per article, showing:</p>
<ul>
<li>Article title</li>
<li>Category</li>
<li>Tags (click to edit)</li>
<li>SEO description (click to edit)</li>
<li>Character count indicator (green = good length, red = too short or too long)</li>
</ul>

<h2>Editing Descriptions in Bulk</h2>
<ol>
<li>Click the description cell for any article</li>
<li>An inline text box appears — type or edit the description</li>
<li>Press Enter or click away to save</li>
<li>The change is saved immediately</li>
</ol>
<p>Good SEO descriptions are 120–160 characters long, include the main keyword the article targets, and clearly explain what the reader will learn.</p>

<h2>Managing Tags</h2>
<p>Click the <strong>Content Tags</strong> tool (also under Content in Toolbox) for full tag management:</p>
<ul>
<li>See all tags in use and how many articles use each</li>
<li>Rename a tag — renames it everywhere it is used automatically</li>
<li>Merge two tags — combine "setup" and "installation" into one tag across all articles</li>
<li>Delete a tag — removes it from all articles</li>
<li>Create a new tag for use in articles</li>
</ul>

<h2>Adding Tags to Articles</h2>
<p>Tags can be added from:</p>
<ul>
<li>The SEO Metadata table (click the tags column for an article)</li>
<li>The article's Inspector panel in the Documentation editor (Tags field)</li>
<li>The Content Tags tool in Toolbox (bulk tag many articles at once)</li>
</ul>

<h2>How Tags Help Readers</h2>
<p>Tags appear below article headings on the public knowledge base. Readers can click a tag to see all other articles with the same tag — this is a great way to show related content beyond just what is in the same category. Tags also help the AI search and recommendation engine surface more relevant results.</p>

<h2>SEO Best Practices</h2>
<ul>
<li>Every article should have a unique, keyword-rich description</li>
<li>Write descriptions as if they will appear in a Google search result snippet (because they might)</li>
<li>Use 3–7 tags per article — too few reduces discoverability, too many dilutes relevance</li>
<li>Be consistent with tag names — use the exact same tag across articles, not variations ("API", "api", "APIs" should all be the same tag)</li>
</ul>
